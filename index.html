How to deploy the dist directory of a Vue+Vite project to gh-pages

1. Configura la base correcta en vite.config.js (como indica https://es.vitejs.dev/guide/static-deploy.html) (Si es para un repositorio tuusuario.github.io no necesita, pero sí, si es cualquier otro repositorio) 2. npm run build 3. git add dist -f 4. git commit -m 'mensaje de commit' 5. git subtree push --prefix dist origin gh-pages 

        import { defineConfig } from 'vite'
          import vue from '@vitejs/plugin-vue'
          
          export default defineConfig({
            base: '/nombredelrepositorio/',
            plugins: [vue()], 
          })

How to deploy the dist directory of a React+Vite project to gh-pages

1. instalar gh-pages (con npm gh-pages ó pnpm gh-pages) 2. agregar dos abreviaciones de script en el package json (predeploy y deploy)

        "scripts": {
          "dev": "vite",
...
          "predeploy": "npm run build",
          "deploy": "gh-pages -d dist"
        },
        "de

Luego, solo necesitas ejecutar pnpm run deploy Posiblemente funcione igual en cualquier proyecto en React, aunque no use Vite.

How to see Vue+Vite project on your phone

On package.json add add '--host' in the line that starts with "dev" 

          "scripts": {
            "dev": "vite --host",
            ...
          },

How to add Alias and port

On package.json 

          // https://vitejs.dev/config/
          export default defineConfig({
            plugins: [vue()],
            // server: {
            //   port: 3001
            // },
            resolve: {
              alias: {
                "@": fileURLToPath(new URL("./src", import.meta.url))
              }
            }
          });
